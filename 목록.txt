
-------------------- 05/05 만든 파일(황자준)


-------- 1차 코드(요약내용)

1. login.xml

--> 앱 실행 시 처음 나타나는 로그인 화면



2. LoginAcitivity.java

--> login.xml 의 기능을 처리해주는 자바파일

   1. 로그인 버튼 클릭 처리  -> 조건 만족 시 MainActivity로 화면 전환

   2. 회원가입 클릭 시 Dialog 보여줌

      -> register.xml을 팝업 형태로 보여줌,  팝업 안에 있는 버튼 클릭 이벤트도 처리함 (register_btn, cancel_btn)



3. register.xml 

--> 회원가입 화면을 구성하는 UI 레이아웃 파일



4. activity_main.xml

--> 앱의 홈 화면 레이아웃이자 전체 구조를 정의하는 메인 UI

   1. DrawerLayout + RecyclerView 조합으로 왼쪽 슬라이딩 메뉴 제공
   2. 상단 툴바를 직접 만들고, 오른쪽에 유저 프로필 표시
   3. 메인 콘텐츠는 Fragment로 교체하면서 구성할 수 있도록 설계됨

5. MainAcitivity.java

--> activity_main.xml과 1:1로 매칭되는 기능 처리용 코드

   1.  앱이 처음 실행될 때만 fragment_container 안에 HomeFragment를 띄움

 	

6. home.xml

--> HomeFragment가 보여주는 화면 레이아웃

    1. MainActivity.java는 실행되자마자 fragment_container에 HomeFragment를 띄워줌.

      HomeFragment는 내부에서 setContentView()가 아닌 onCreateView()에서 home.xml을 불러와 화면을 구성함.


7. HomeFragent.java

--> 	MainActivity는 큰 틀을 잡고, HomeFragment는 **본문 화면(콘텐츠)**을 맡는 구조로 모듈화를 위함

   1. 연결 흐름

 --- > LoginActivity ➝ 로그인 성공 시 ➝ MainActivity ➝
MainActivity의 onCreate에서 ➝ getSupportFragmentManager().beginTransaction() ➝
HomeFragment 붙이기 ➝ home.xml 레이아웃 UI가 화면에 보임

--------------> 요약

1. 앱 실행 흐름

 앱 시작 → LoginActivity 실행 → 로그인 성공 → MainActivity 실행 → HomeFragment UI 표시

==== 

🧱 주요 구성 요소
✅ 1. LoginActivity (로그인 화면)
연결된 XML: login.xml

기능:

아이디, 비밀번호 입력

로그인 버튼 클릭 시 유효성 검사 (admin / 1234)

성공하면 MainActivity로 이동

하단 회원가입 TextView 클릭 시 → register.xml을 다이얼로그로 보여줌

✅ 2. register.xml (회원가입 UI 다이얼로그)
이름, 아이디, 비밀번호, 이메일 등을 입력하는 폼

아직 DB 연동은 없고, register_btn, cancel_btn 버튼 이벤트 처리만 있음

✅ 3. MainActivity (앱 메인화면)
연결된 XML: activity_main.xml

툴바 포함, 왼쪽 사이드 메뉴 포함 (RecyclerView)

fragment_container에 HomeFragment를 동적으로 붙임

✅ 4. HomeFragment (홈 콘텐츠 영역)
연결된 XML: home.xml

약속 카드뷰, 버튼 3개:

약속 만들기 (btnCreateMeeting)

약속 목록 보기 (btnMeetingList)

지도 보기 (btnMap)

아직 클릭 이벤트는 구현 안됨
